# 审计报告：各品牌相机连接 / 识别 / 显示 / 语音播报全链路分析

> 审计时间：2026-04-24
> 项目路径：D:\BulkMaterialsForm_v2\BulkMaterialsForm_v2

---

## 一、架构总览

软件同时支持 **4 套相机品牌 × 2 种大屏协议 × 2 种控制卡** 的组合，通过 `tb_Device.motherboardType`（控制卡类型）和 `MainData.DPLX`（大屏类型）两条维度路由到不同实现。

```
相机识别回调（4个独立UserControl）
    │
    ├─ UserControlZS   (臻识 VzClientSDK)   ──┐
    ├─ UserControlDH    (大华 NetSDK)         │
    ├─ UserControlHaiKang (海康 CHCNetSDK)  ├─→ 统一 VehicleNoInfoView
    └─ UserControlHX    (华夏 ICE IPC SDK)   ──┘
                                                   │
                                                   ▼
                         ┌─────────────────────────────────────┐
                         │          大屏显示 (DPLX)            │
                         │  YB(YB_SDK) │ ZH(ZHSDK) │ FK(FKTcp) │
                         └─────────────────────────────────────┘
                                                   │
                                                   ▼
                         ┌─────────────────────────────────────┐
                         │       控制卡显示/语音 (FK / KF)      │
                         │  FK: 方控协议(LED+语音)              │
                         │  KF: 科发协议(LED+语音+红绿灯)        │
                         └─────────────────────────────────────┘
```

## 二、相机品牌详细审计

### 1. 臻识（ZS）— UserControlZS.cs

| 维度 | 内容 |
|------|------|
| **SDK** | `VzClientSDK`（C DLL 封装） |
| **初始化** | `VzLPRClient_Setup()` → `Sdk_ZhenShi_init` 全局单次 |
| **连接** | `VzLPRClient_Open(IP, 80, "admin", "admin")` 硬编码端口/账号 |
| **预览** | `VzLPRClient_StartRealPlay(_logHandle, videoHandle)` |
| **回调注册** | `VzLPRClient_SetPlateInfoCallBack` 车牌回调 + `VzLPRClient_SetGPIORecvCallBack` IO 回调 |
| **485透传** | `VzLPRClient_SerialStart` 开启串口，`VzLPRClient_SerialSend` 发数据 |
| **抬杆** | GPIO IO 触发 → `OnserialRECV` → `barrierGate(nVal)` |
| **图片** | `VzLPRClient_ImageSaveToJpeg` 保存全景和车牌特写 |

**问题**：
- `VzLPRClient_Open` 账号密码硬编码为 `admin/admin`，端口硬编码为 `80`，无法从配置读取
- 连接失败只 `setButtonEnable(false)` 返回，无重连机制

---

### 2. 大华（DH）— UserControlDH.cs

| 维度 | 内容 |
|------|------|
| **SDK** | `NetSDKCS`（大华 .NET SDK） |
| **初始化** | `NETClient.Init()` → `Sdk_DaHua_init` 全局单次 |
| **连接** | `LoginWithHighLevelSecurity(IP, 37777, 账号, 密码, TCP)` |
| **预览** | `NETClient.StartTalk` 开始对讲/实时流 |
| **回调** | `fAnalyzerDataCallBack` 交通事件回调（车牌、车颜色） |
| **抓拍** | `SnapPictureToFile` 三次重试，写入数据库三张图片路径 |
| **抬杆** | `NETClient.ControlDevice(_logHandle, OPEN_STROBE)` |
| **485透传** | `Send485()` 方法体**完全为空**，DH 品牌无 485 透传实现 |
| **大屏** | 通过 `DPLX` 判断走 YB_SDK 或 ZHSDK 或 FKTcp |
| **语音** | 内置大华 `NET_BROADCAST_INFO` 广播，无外部 TTS |

**问题**：
- `Send485` 为空方法，如果配置了 FK/KF 控制卡则完全失效
- 抓拍失败重试逻辑中 `Thread.Sleep(snapNumber * 1000)` 依赖配置参数，未做上限保护
- `fAnalyzerDataCallBack` 使用字符串比较判断车牌颜色（"Blue"等英文），颜色映射存在遗漏颜色

---

### 3. 海康（HaiKang）— UserControlHaiKang.cs

| 维度 | 内容 |
|------|------|
| **SDK** | `CHCNetSDK`（海康 SDK C DLL 封装） |
| **初始化** | `CHCNetSDK.NET_DVR_Init()` → `Sdk_HaiKang_init = true`，全局单次 |
| **连接** | `NET_DVR_Login_V40(IP, 8000, 账号, 密码)` 同步登录 |
| **预览** | `NET_DVR_RealPlay_V30` 视频流 |
| **回调** | `SERIALDATACALLBACK` 串口数据回调 |
| **抓拍** | `NET_DVR_CaptureJPEGPicture` 三次重试 |
| **抬杆** | 无直接 SDK 抬杆，通过 `Operation` 委托调用外部 |
| **485透传** | `NET_DVR_SerialSend(m_SerialStart, 1, data, len)` 通过通道1 |
| **录像** | `ExchangeBoard.Download` 使用 `NET_DVR_GetFileByTime_V40` |
| **语音** | Windows `SpeechSynthesizer` TTS，朗读 `PlayText` |

**问题**：
- 录像机端口硬编码为 `"8000"`（`short.Parse("8000")`），无法从配置读取
- 海康 SDK 的登录密码明文记录在日志中（`LogSave.HKLog`）
- 重连机制：每 60 秒检查一次 `IsEnd`，断开后自动重连（`videoClientOpen_HaiKang`）

---

### 4. 华夏/ICE（HX）— UserControlHX.cs

| 维度 | 内容 |
|------|------|
| **SDK** | `ipcsdk`（`.\HX\ice_ipcsdk.dll`，第三方封装） |
| **初始化** | `ICE_IPCSDK_Init()` 初始化 SDK |
| **连接** | `ICE_IPCSDK_OpenDevice_Passwd(IP, password)` 打开设备 |
| **预览** | `ICE_IPCSDK_OpenPreview` 或 `ICE_IPCSDK_Open` |
| **回调** | `ICE_IPCSDK_OnPlate` 车牌回调，`ICE_IPCSDK_OnIOEvent` IO 事件回调 |
| **抬杆** | `ICE_IPCSDK_OpenGate(_logHandle)` |
| **485透传** | `ICE_IPCSDK_TransSerialPort(_logHandle, data, len)` |
| **语音** | FK/KF 控制卡自带语音，无独立 TTS |

## 三、控制卡（主板）详细审计

### 1. FK 方控协议 — FKTcp.cs + LED.cs

| 维度 | 内容 |
|------|------|
| **通信** | TCP Socket 直连控制卡 IP，端口 **8800**，每次发包新建连接 |
| **协议** | 自定义二进制协议：帧头 `AA 55` + 数据 + CRC16 + 帧尾 `AF` |
| **显示** | `GetLedTextByte(text, 行号)` 构造显示命令，最多 4 行 |
| **语音** | `FangKongStandardVoiceByte(车牌号, cmdList)` 语音合成，cmdList传指令码（40=允许，43=禁止） |
| **车牌大屏** | `FKTcp.Send()` 只支持 `{车牌号}` `{排放阶段}` `{燃油类型}` `{通行状态}` `{验证提示}` 五个占位符 |

**问题**：
- 每次 `Send` 都新建 TCP 连接，无连接池，高频调用时效率低
- Socket `Connect` 和 `Close` 无异常处理详细信息，`catch` 空块
- FK 大屏占位符**不支持** GL 管控相关字段（无 `{预警等级}` `{管控策略}` 等）

---

### 2. KF 科发协议 — LED.cs

| 维度 | 内容 |
|------|------|
| **通信** | 通过相机 RS485 透传（`Send485`），不独立 TCP 连接 |
| **协议** | 科发自定义二进制协议：`地址+100+FF+FF+功能码+长度+数据+CRC` |
| **显示** | `ShowKeFa1TextByID(TextID, 文本, 地址, 次数)` 显示指定行 |
| **语音** | `KeFa_SoundPlay(地址, 文本)` 语音合成，支持"车牌号欢迎光临"等 |
| **红绿灯** | `KeFa_LedLampLv` 绿灯，`KeFa_LedLampLan` 蓝灯（实际代码注释为蓝灯但变量名暗示绿灯） |
| **出口语音** | "车牌号一路顺风"，入口："车牌号欢迎光临"，禁止："车牌号禁止通行" |

## 四、大屏协议审计

### 1. YB 异步大屏 — YB_SDK.cs

| 维度 | 内容 |
|------|------|
| **硬件** | 仰邦（BX）系列 LED 控制卡 |
| **SDK** | `Led5kSDK`（DLL 封装），连接端口固定 `5005` |
| **动态区** | `SCREEN_SendDynamicArea` 临时显示，关联车牌通行结果 |
| **静态区** | `Led5kstaticArea` 固定内容区域，定时刷新 |
| **占位符（现支持）** | `{车牌号}` `{排放阶段}` `{燃油类型}` `{通行状态}` `{验证提示}` `{预警等级}` `{管控策略}` `{响应等级}` `{企业名称}` `{管控开始时间}` `{管控结束时间}` `{管控更新时间}` `{当前时间}` |
| **空值兜底** | 全部 GL 管控字段有 `string.IsNullOrWhiteSpace` 兜底处理 |

**问题**：
- 动态区/静态区占位符检测条件必须精确匹配（漏掉任何一个占位符就不进入动态区逻辑）
- 颜色控制 `\C1` `\C2` `\C3` 协议指令需紧贴文本前端，无其他颜色选项

---

### 2. ZH 臻识大屏 — ZHSDK.cs

| 维度 | 内容 |
|------|------|
| **硬件** | 臻识 LED 屏 |
| **SDK** | `LedAgent.ShowInstantText` |
| **占位符（现支持）** | `{车牌号}` `{排放阶段}` `{燃油类型}` `{通行状态}` `{验证提示}` `{预警等级}` `{管控策略}` `{响应等级}` `{企业名称}` `{管控开始时间}` `{管控结束时间}` `{管控更新时间}` |
| **空值兜底** | 全部 GL 管控字段有 `string.IsNullOrWhiteSpace` 兜底处理 |

## 五、语音播报方式汇总

| 使用位置 | 语音方式 | 品牌关联 |
|----------|----------|----------|
| `UserControlDH` 验证结果 | 大华内置 `NET_BROADCAST_INFO` 广播 | DH |
| `UserControlHaiKang` 手动抬杆 | Windows TTS `SpeechSynthesizer` | HK |
| `UserControlHX` / `UserControlZS` / `UserControlQY` | FK/KF 控制卡自带语音（`FangKongStandardVoiceByte` / `KeFa_SoundPlay`） | FK / KF |
| 全局电脑语音播报 | `MainData.dnyybb` 开关，`WindowsXFTTSUtils`（讯飞离线 TTS DLL） | 全局 |

## 六、核心问题汇总

| 优先级 | 问题 | 影响范围 | 文件 |
|--------|------|----------|------|
| 🔴 高 | `Send485` 空方法 | DH 品牌控制卡完全失效 | `UserControlDH.cs` |
| 🔴 高 | FK 大屏不支持 GL 管控占位符 | 高凌项目大屏无法显示预警/策略 | `FKTcp.cs` |
| 🟡 中 | 臻识登录账号密码硬编码 `admin/admin` | 无法对接需要认证的设备 | `UserControlZS.cs` |
| 🟡 中 | 海康/华夏录像机端口硬编码 `8000` | 不支持非标准端口设备 | `HKSDK.cs` / `ipcsdk.cs` |
| 🟡 中 | FK 控制卡每次发包新建 TCP 连接 | 高频场景性能差、连接不稳定 | `FKTcp.cs` |
| 🟡 中 | DH 颜色映射缺失多种颜色 | 部分车牌颜色显示为"其他" | `UserControlDH.cs` |
| 🟢 低 | FK 大屏只支持 5 个占位符 | YB/ZH 已支持 12 个，FK 落后 | `FKTcp.cs` |
| 🟢 低 | 日志明文记录密码 | 安全审计风险 | `HKSDK.cs` |

## 七、数据流验证（以一次完整通行判断为例）

```
车辆触发
    │
    ▼
相机品牌回调 (ZS/DH/HK/HX)
    │ 车牌号 / 颜色 / 图片
    ▼
平台验证 (DJPT 平台)
    │ 超低排放 / 高凌管控 / 天地人车 / 信阳 / 安车 / 开封V1
    ▼
TrafficStatus + ExeLog + fuelType + emissionStandard 填充
    │
    ├─→ 大屏显示（YB/ZH/FK）
    │       ├─ YB_SDK.YBLedSendMes   (12个占位符) ✅
    │       ├─ ZHSDK.SendZH          (12个占位符) ✅
    │       └─ FKTcp.Send            (5个占位符) ⚠️ 无GL字段
    │
    └─→ 控制卡显示+语音 (FK/KF)
            ├─ FK: 方控协议 TCP→8800
            │       ├─ 通行: 40号语音 + 4行LED ✅
            │       └─ 禁止: 43号语音 + 4行LED ✅
            │
            └─ KF: 科发协议 RS485透传
                    ├─ 通行: 语音+绿灯 ✅
                    └─ 禁止: 语音+红灯 ✅
```
